photinus

Mesh monitoring with no center.

Every host runs a lantern. The lanterns watch each other, gossip what they see, and alert only when enough of the swarm agrees. There is no central collector, no dashboard host, and nothing whose failure makes the monitoring blind.

Named after Photinus carolinus, the firefly that flashes in unison with nothing coordinating it.

How it holds together

  • No single point of failure. No primary, no coordinator, no seed that matters after startup. Any lantern can vanish and the swarm keeps watching.
  • Alerts fire on agreement, not opinion. One lantern seeing a host down is a fact about that lantern. Quorum is what makes it an outage.
  • Partitions go quiet, not loud. Quorum counts against the swarm as last known, so a cut-off minority cannot page you about the majority it lost.
  • Status is always local. Ask any lantern what the swarm sees; it answers from its own memory, even with the network on fire.
  • One static binary. Drop it on the host and run it. No runtime, no dependencies, small boxes welcome.

What it looks like

# on every host
photinus run -seed lantern-a:7946 -watch tcp:db:5432

# on any host, any time
photinus status

tcp   db:5432   DOWN  (5 of 7 lanterns agree, quorum is 4)

Pre-alpha. The first lanterns are being built in the open.

photinus is not a metrics store, not agentless, and not for a single server. It answers one question: is it broken right now, and does the swarm agree.
